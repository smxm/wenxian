# 正式报告模块标题展示规则

本文档用于固化正式报告模块中的文献标题展示约定，避免后续在调整 prompt、模板或渲染器时，重新退回“默认展示中文译名”的行为。

## 一、总体原则

正式报告中所有面向最终用户的文献标题展示，默认以“原标题”为准，而不是以中文译名为准。

这样处理的原因是：

- 用户需要将报告中的文献标题与数据库导出记录、原始 BibTeX / ENW 文件和检索结果一一对应。
- 对英文文献而言，原标题通常是英文标题；若在关键位置改为中文译名，会增加检索和匹配成本。
- 对中文文献而言，原标题本身就是中文标题，直接显示即可。

## 二、数据结构约定

在 `LiteratureCard.source_record` 中，标题字段应按以下方式理解：

- `title_en`
  - 作为正式报告中的默认展示标题。
  - 如果原始文献为英文文献，则这里必须保存英文原标题。
  - 如果原始文献为中文文献，则这里可直接保存中文原标题。

- `title_zh`
  - 用于保存中文标题或中文译名。
  - 对中文文献而言，可以与 `title_en` 相同。
  - 对英文文献而言，可以保存自然、准确的中文译名。
  - 该字段不作为正式报告中的默认标题来源。

## 三、正式报告中的使用规则

以下位置必须统一显示原标题：

- 分类整理中的“相关代表文献包括”
- “重点推荐文献”中的各篇标题
- “文献逐篇整理”中的每篇主标题
- “附录：文献信息一览”表格中的标题列

以下位置可以显示中文题目，但仅作为辅助信息：

- “文献逐篇整理”正文中的“中文题目”一行

补充约束：

- 只有当 `title_zh` 存在，且其内容与原标题不同的时候，才在逐篇整理正文中额外显示“中文题目”。
- 如果文献本身就是中文文献，则不应在主标题下重复再写一遍相同的中文标题。

## 四、对 prompt 的要求

为保证这一规则能够稳定执行，文献卡片生成 prompt 与正式报告概览 prompt 都需要遵循以下要求：

- 不得将英文原标题改写为中文标题并写入 `title_en`。
- 不得将中文文献强行翻译为英文标题后写入 `title_en`。
- 概览生成阶段只输出 `paper_id`，不要在该阶段自由重写代表文献标题。
- 最终标题展示由渲染器统一根据 `title_en` / `title_zh` 规则处理。

## 五、实现要求

渲染器和后续模板实现应满足以下稳定约束：

- 标题显示逻辑必须集中在统一函数中维护，避免不同章节各自拼接标题。
- 默认标题函数应优先返回原标题。
- 中文题目显示应作为单独的辅助函数处理，而不是与默认标题函数混用。
- 后续若扩展 PDF、Word 或 Web 页面导出，应沿用同一套标题展示规则。

## 六、结论

正式报告模块的标题展示规则可以概括为：

“默认显示原标题，中文题目仅作辅助展示。”

这是一条稳定交付规则。后续无论是修改 prompt、调整 Markdown 模板，还是扩展其他导出格式，都不应改变这一原则。
