# 简洁报告工作流说明

本模块当前默认采用“简洁整理报告”工作流，用于在初筛完成后，基于纳入文献生成一份适合直接阅读和转发的中文整理稿。

## 适用范围

- 适合在初筛之后快速形成一版对外可读的文献整理结果
- 适合先产出一版结构清楚的总结稿，再决定是否做更深的专题综述
- 不再承担主项目中的初筛职责，初筛仍由主项目完成

## 默认报告结构

1. 相关文献总体情况
2. 类型划分
3. 逐篇文献总结分析

其中第三部分固定为：

- 文献原标题作为标题
- `总结：`
- `分析：`

## 推荐使用方式

优先从已有初筛输出目录生成报告，例如：

```bash
python E:\wenxian\literature_screening\separated_modules\formal_report_module\scripts\run_report.py ^
  --screening-output-dir E:\wenxian\literature_screening\data\output\run_robot_screening_deepseek ^
  --report-output-dir E:\wenxian\literature_screening\separated_modules\formal_report_module\outputs\run_robot_simple_report_deepseek ^
  --project-topic "地下钻进机器人推进与破土机制" ^
  --provider deepseek ^
  --model-name deepseek-chat ^
  --api-base-url https://api.deepseek.com/v1 ^
  --api-key-env DEEPSEEK_API_KEY
```

## 默认输出文件

- `literature_report.md`
  - 默认成品文件
- `simple_report.md`
  - 兼容历史命名
- `paper_notes.json`
  - 每篇文献的结构化总结和分析

## 设计原则

- 以“可直接阅读的整理稿”为目标，而不是展示过程参数
- 标题尽量保留原标题，便于与原文献一一对应
- 总结与分析分开，避免把摘要转述和主题价值判断混在一起
- 失败时允许回退到本地兜底逻辑，优先保证报告可以产出
