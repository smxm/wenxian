## Current Product Shape

- `/threads/new` now only creates a thread from the research need; follow-up work starts after the thread exists.
- The thread detail page is the main control surface: one edit entry at the top, plus four stage actions for search strategy, screening, unified review, and report generation.
- The unified review area is a project-level candidate workbench with separate round context, stable candidate identities, batch/single-item status updates, and report-source decisions driven by final inclusion.
- The report stage is always visible on the thread page, even when the included-record count is `0`, and now lets the user choose provider/model credentials for generation.
- Project-managed `report_source` / `fulltext_ready` datasets are rebuilt before report submission, so report generation can pick up refreshed workbench selections and corrected bibliography metadata.
- Report-source RIS exports now preserve `volume` / `number` / `pages`, including fallback extraction from legacy `raw_bibtex`, so GB/T 7714 references can keep page information when the source data already contains it.
- `deepseek-reasoner` report jobs now enforce a higher token floor and clearer empty-content errors, while the simple-report fallback taxonomy now handles Chinese humanities/literature topics instead of collapsing them into a single generic mechanism category.
- Tencent Cloud deployment is documented around a tarball-upload plus `deploy/server-update.sh` flow, with runtime data and Basic Auth moved out of `/opt/wenxian` into persistent directories so updates do not wipe threads or reports.
